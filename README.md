# 🌍 Drifter — Unified Terminal Starter Kit

Единая и предсказуемая среда командной строки для **Windows Terminal, WSL и удалённых серверов (SSH)**.

## 🎯 Проблематика и Решение

**Проблема:** Отсутствие единообразия между локально настроенным терминалом и удаленными средами по SSH или в контейнерах. Зачастую переход на удаленный сервер сопровождается потерей автодополнений, привычных алиасов и визуальной информативности консоли.

Данный проект решает инфраструктурную задачу доставки профессионального рабочего окружения с соблюдением принципов отказоустойчивости и безопасности:

1. **Bash как фундамент:** Вместо изменения системной оболочки на `zsh` или `fish` (что часто нарушает работу `cron` и системных сценариев), производится расширение стандартного `bash`. С помощью фреймворка `ble.sh` и утилиты `starship` стандартная оболочка получает функционал современных эмуляторов терминала.
2. **Адаптивная деградация (Fallback):** Проект использует современные Rust-утилиты (`eza`, `bat`, `zoxide`). В средах, где их установка невозможна (старые серверы), шаблонизатор Chezmoi автоматически откатывает конфигурации на безопасные POSIX-эквиваленты (`ls`, `cat`), предотвращая ошибки синтаксиса `command not found`.
3. **Недеструктивная установка:** Сценарии инициализации не удаляют существующие файлы конфигурации пользователя. Предыдущие версии `~/.bashrc` и аналогичных файлов архивируются и аккуратно переименовываются в `.legacy`. При этом их функционал продолжает выполняться "под капотом".
4. **Интеграция Fish (Опционально):** Для локальных рабочих сред (например, WSL) предусмотрена опциональная активация интерактивной оболочки `Fish`, использующей единую с `bash` базу переменных окружения (`env.list`). Данный подход предотвращает дублирование кода.

## 🧱 Технологический стек (Overview)
- **Chezmoi** — ядро системы (управление dotfiles, шаблонизация конфигов под спецификации железа).
- **Bash + Ble.sh** — отказоустойчивая база с умным вводом (Bash Line Editor).
- **Starship** — быстрый кроссплатформенный генератор промпта (на чистом Rust).
- **Modern CLI** — высокопроизводительные утилиты (`eza`, `bat`, `zoxide`, `fd`).
- **FZF** — сверхбыстрый Fuzzy-поиск с динамическим предпросмотром кода в терминале.

## ⚡ Быстрый старт

Предварительно загрузите и установите шрифт **[JetBrainsMono Nerd Font](https://www.nerdfonts.com/font-downloads)** в настройках используемого терминала (потребуется для корректного отображения иконок Starship).

Выберите подходящий сценарий развертывания:

### 👑 Установка одной командой (Bare Metal / Чистая машина)
```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/Ephraimus/chezmoi/main/scripts/install.sh)"
```

### 💻 Локальная установка (Если репозиторий уже клонирован)
```bash
cd ./scripts
./bootstrap.sh
```

### 🐠 Локальная установка + активация Fish (Для Desktop / WSL)
```bash
cd ./scripts
./bootstrap.sh --use-fish --set-login-shell fish
```

### ☁️ Удаленная установка по SSH (Remote Deployment)
```bash
cat ./scripts/bootstrap-remote.sh | ssh user@host 'bash -s -- --repo-url https://github.com/Ephraimus/chezmoi.git --repo-branch main'
```

## 🛠 Ключевые файлы конфигурации

Модификация переменных и алиасов производится централизованно через механизм локального хранилища Chezmoi.

| Файл                               | Назначение                           | Пример                         |
| ---------------------------------- | ------------------------------------ | ------------------------------ |
| `dot_config/shell/env.list`        | Универсальные переменные окружения   | `EDITOR=vim`                   |
| `aliases.sh.tmpl`                  | Единый список алиасов (Chezmoi)      | `alias gs='git status -s'`     |
| `~/.config/chezmoi/chezmoi.toml`   | Специфичные настройки целевой машины | `use_fish = false`             |

## 🔄 Синхронизация изменений на удалённые машины

После внесения изменений в конфигурационные файлы и публикации коммита в Git, для обновления целевых серверов доступны два метода:

**На одной машине (вручную):**
```bash
chezmoi update
```

**На всех сконфигурированных серверах сразу:**
```bash
./scripts/sync-all-remotes.sh
```
Предварительно необходимо добавить адреса серверов в массив `HOSTS` внутри скрипта. Требуется SSH-аутентификация по ключам (без пароля). Подробности — в [ARCHITECTURE.md](ARCHITECTURE.md#8-синхронизация-удалённых-хостов).

## ❓ Частые вопросы (FAQ)

<details>
<summary><b>1. Почему вместо иконок отображаются квадраты или символы ошибки?</b></summary>
Для корректного отображения иконок необходим пропатченный шрифт. Загрузите и установите шрифт из коллекции <a href="https://www.nerdfonts.com/font-downloads">Nerd Fonts</a> (рекомендуется <i>JetBrainsMono Nerd Font</i>) и выберите его в качестве шрифта по умолчанию в настройках эмулятора терминала.
</details>

<details>
<summary><b>2. Как отключить тяжеловесный автокомплит `ble.sh` на серверах с ограниченными ресурсами?</b></summary>
Отредактируйте локальный файл <code>~/.config/chezmoi/chezmoi.toml</code>, установив параметр <code>use_blesh = false</code>. Выполните команду <code>chezmoi apply</code> для перегенерации файлов инициализации и отключения фонового анализатора синтаксиса `ble.sh`.
</details>

<details>
<summary><b>3. Как корректно вносить изменения в алиасы и переменные?</b></summary>
Редактирование следует производить через шаблонизатор. Используйте команду <code>chezmoi edit ~/.config/shell/aliases.sh</code>. Сохраните изменения и обязательно выполните <code>chezmoi apply</code>, чтобы шаблонизатор пересобрал файл с предварительной проверкой доступности системных утилит.
</details>

<details>
<summary><b>4. Как полностью удалить пайплайн конфигураций и вернуть систему в исходное состояние?</b></summary>
Сценарии установки не удалили оригинальные конфигурации. Для полного удаления запустите скрипт аппаратного сброса: <code>./scripts/hard-reset-shell-configs.sh</code>. Данное действие удалит файлы симлинков, сгенерированные пайплайном. Затем скопируйте резервные оригинальные файлы (<code>.bashrc</code>, <code>.zshrc</code> и прочие) из директории <code>~/.shell-migration-backup/</code> обратно в корневую папку <code>~/</code> текущего пользователя.
</details>
