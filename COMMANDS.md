# Справочник Команд

Краткий справочник по алиасам и shell-функциям, которые ставит Drifter.

## Что Где Работает

- **Bash / Zsh**: доступны алиасы из `aliases.sh` и функции из `common.sh`.
- **Fish**: доступны только shell-специфичные части и сгенерированные алиасы; POSIX-функции вроде `mkd`, `dpsg`, `dlog`, `dsh`, `dbash`, `dins` сюда не входят.
- **PowerShell**: отдельный справочник находится в [POWERSHELL.md](POWERSHELL.md).

## Самое Частое

- `gs` — быстро проверить состояние git-репозитория
- `glog` — посмотреть компактную историю коммитов
- `mkd <dir>` — создать директорию и сразу перейти в неё
  `mkd tmp/project`
- `dcud` — поднять compose-проект в фоне
- `dclf` — смотреть логи compose-проекта
- `dpsg [pattern]` — найти контейнер по имени или short ID
  `dpsg api`
- `dbash <container>` — зайти в контейнер через `bash`, с fallback на `sh`
  `dbash backend-api-1`
- `myip` — показать текущий публичный IP

## Навигация

- `..` — перейти на уровень выше
- `...` — перейти на два уровня выше
- `mkd <dir>` — создать директорию и сразу перейти в неё
  `mkd tmp/project`

## Git

- `gs` — проверить текущее состояние рабочей директории
- `ga` — добавить изменения в staging
  `ga .`
- `gc` — создать коммит
  `gc -m "Update config"`
- `gca` — исправить последний коммит
- `gp` — отправить изменения в удалённый репозиторий
- `gl` — подтянуть изменения из удалённого репозитория
- `gco` — переключить ветку или восстановить файл
  `gco main`
- `gb` — посмотреть или управлять ветками
- `gd` — показать незакоммиченные изменения
- `glog` — компактная история git с графом

## Docker Compose

Эти команды работают на уровне compose-проекта и ожидают, что ты находишься в каталоге, где `docker compose` видит нужный compose-файл.

- `dc` — короткий `docker compose`
  `dc ps`
- `dcud` — поднять compose-проект в фоне
- `dcd` — остановить и удалить compose-сервисы
- `dclf` — смотреть логи compose-проекта

## Docker Контейнеры

Эти команды работают на уровне отдельных контейнеров. В качестве аргумента можно использовать имя контейнера или short ID.

- `d` — короткий `docker`
  `d ps`
- `dps` — список запущенных контейнеров
- `dlogf` — follow логов контейнера
  `dlogf backend-api-1`
- `dex` — интерактивный `docker exec -it`
  `dex backend-api-1 sh`
- `dpsg [pattern]` — показать контейнеры с short ID, именем, image и status; при переданном шаблоне фильтровать по подстроке
  `dpsg api`
- `dlog <container>` — смотреть логи контейнера по имени или short ID
  `dlog backend-api-1`
- `dsh <container>` — зайти в контейнер через `sh`
  `dsh backend-api-1`
- `dbash <container>` — зайти в контейнер через `bash`, с fallback на `sh`
  `dbash backend-api-1`
- `dins <container>` — выполнить `docker inspect`
  `dins backend-api-1`

## Файлы И CLI-Утилиты

Ниже перечислены алиасы, итоговый вид которых зависит от доступных утилит на конкретной машине.

- `ls` — `eza --icons=auto --color=auto`, если установлен `eza`
- `ll` — подробный список файлов; через `eza` или fallback на `ls -lah`
- `la` — список включая скрытые файлы; через `eza` или fallback на `ls -A`
- `l` — компактный список по колонкам / с индикаторами; через `eza` или fallback на `ls -CF`
- `tree` — дерево директорий через `eza`, если `eza` установлен
- `cat` — `bat --paging=never` или `batcat --paging=never`, если доступны
  `cat README.md`
- `fd` — alias на `fdfind` в Debian/Ubuntu окружениях, где `fd` называется иначе
  `fd docker`
- `v` — быстрый запуск `vim`
  `v ~/.bashrc`
- `c` — очистить терминал
- `myip` — показать текущий публичный IP

## Примечания

- Если какая-то команда из списка отсутствует, сначала проверь shell: часть команд доступна только в Bash/Zsh.
- Некоторые file/CLI aliases зависят от наличия утилит `eza`, `bat`, `batcat`, `fd` или `fdfind`.
- После обновления шаблонов или алиасов применяй изменения через:
  `chezmoi apply`
